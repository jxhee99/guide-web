Agent 사용자 가이드를
Claude Design으로 만들기

AI 포탈의 Agent 사용자 가이드 HTML 페이지를 Claude Design에게 그리게 하는 작업 가이드.
파일 준비 → 콘텐츠 작성 → 프롬프트 → 검토 4단계로 정리했습니다.

시작하기 워크플로우 보기
⭐
이 가이드의 가장 중요한 약속 2가지
Claude Design에게 작업을 시킬 때 항상 이 두 가지를 충족해야 합니다
01
약속 #1

디자인 가이드
(.md) 일관 사용

모든 사용자 가이드는 같은 SHINHAN_CLAUDE_DESIGN_GUIDE.md를 첨부합니다.
컬러·폰트·레이아웃을 임의로 바꾸지 말고, 가이드 토큰만 사용하세요.

✅ --shinhan-blue 같은 토큰 사용 ❌ #0046FF 직접 하드코딩
02
약속 #2

가이드에 필요한
이미지 + 설명 필수

이미지와 설명은 하나의 세트로 제공되어야 합니다. 각 Step은 이미지 파일명 + 해당 화면의 사용자 동작 설명이 쌍으로 들어가야 완성도 있는 가이드가 됩니다.

✅ 이미지 1장 → 설명 1개 대응 ✅ 관련 이미지 여러 장 → 1개 설명으로 묶을 수 있음
⛔ 절대 규칙 (어기면 가이드 결과물 reject)
위반된 결과물은 시안 자체가 reject — 처음부터 다시 그려야 함
① 원신한 폰트 3종만
Pretendard / Inter / Noto Sans 등 외부 폰트 금지
② 디자인 토큰만 사용
컬러·간격·radius 하드코딩 금지
③ 이미지 cover 금지
반드시 object-fit: contain
④ 폰트 굵기 3종만
300 / 500 / 700 — 중간 굵기 금지
⑤ 흰색 로고 사용
네이비 헤더에서 시인성 확보
⑥ 모바일/PC 반응형
640px / 768px 분기점 준수

Agent Design Guide 생성
5개 파일과 에이전트 이름 · URL · 콘텐츠를 추가하면 Claude가 바로 HTML 가이드 페이지를 생성합니다.

🗺️ 사용 방법

5개 파일을 첨부하고, 한 번의 프롬프트로 에이전트 이름 · URL · 콘텐츠 MD를 전달하면 HTML 가이드가 생성됩니다.
해당 파일들은 공유 드라이브 > 개인폴더 > 김주희에서 다운 받으실 수 있습니다.

디자인 가이드 + 폰트 3종 + 로고 + 스크린샷 모두 한 곳에 모읍니다
필수
구분 파일 설명

필수 shinhan_logo.svg 헤더 로고 — 흰색 버전 권장 (네이비 헤더에 표시)

콘텐츠 작성 — Step별 포맷으로 정리
이미지 파일명 / 설명 텍스트
필수
  • 이미지 파일명 — 한 줄에 하나. 관련 이미지가 여러 장이면 쉼표로 나열 가능
  • 설명 텍스트 — 사용자 동작 기준으로 간결하게 (1~3문장)
  • 이미지 + 설명 = 1개 Step — 이 세트를 외로 하나씩 나열하면 Claude가 자동으로 Step으로 인식

3
검토 · 다듬기 — 자주 발생하는 누락 점검
최종
  • 모바일 베젤(.frame-mobile)에 세로형 이미지가 정상 표시되는지
  • 브라우저 베젤(.frame-browser)에 가로형 이미지가 정상 표시되는지
  • 카드 hover 시 translateY(-6px) + 그림자 강조가 작동하는지
  • 형광펜(.highlight, .highlight-warn)이 핵심 키워드에 적용됐는지
  • Phase 네비게이션과 태그(필수/정보/완료)가 자동으로 부여됐는지

💬 프롬프트 작성법 (좋은 예시 vs 나쁜 예시)

Claude Design은 명확한 명세서·짧은 지시문에 가장 잘 동작합니다. 시간 낭비를 피하는 프롬프트 패턴.

🎯 프롬프트 작성 체크포인트
  • 디자인 가이드 MD 항상 첨부 — 프롬프트 본문은 짧게, 디테일은 가이드가 담당
  • 이미지는 파일명만 올릴 때만 타입 명시 — 실 이미지를 올리면 Claude가 비율로 자동 판단
  • "화려하게", "예쁘게", "트렌디하게" 같은 모호한 형용사 금지 — 디자인 가이드가 미적 결정 책임
  • 수정 요청은 구체적으로 — STEP 번호, 무엇이 잘못, 어떻게 고쳐야 하는지 명시

⛔ DO / DON'T — 자주 발생하는 실수

업로드 전 반드시 점검할 항목. 하나라도 어기면 결과물이 가이드 규격에서 벗어납니다.
✓DO
폰트 3종을 모두 업로드 — Light / Medium / Bold 중 하나라도 빠지면 굵기 차이가 사라져 위계가 무너집니다.
✕DON'T
다른 폰트로 대체 금지 — Pretendard, Inter, Noto Sans 등을 쓰면 한·영 균형이 깨집니다. 원신한만 사용.
✓DO
실제 이미지를 함께 업로드하면 Claude가 비율을 보고 portrait/landscape를 자동 판단합니다. 파일명만 적을 때만 (portrait) 명시.
✕DON'T
object-fit: cover 사용 금지 — 이미지가 잘려 콘텐츠를 볼 수 없습니다. 항상 contain.
✓DO
로고는 흰색 SVG 버전을 권장 — 헤더가 네이비 배경이므로 대비가 명확합니다.
✕DON'T
디자인 가이드 MD에 적힌 컬러·radius·간격 값을 임의로 변경하지 마세요 — 토큰 일관성이 깨집니다.
